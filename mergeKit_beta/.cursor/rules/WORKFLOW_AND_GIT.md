# 开发工作流与 Git（mergeKit_beta）

## 变更范围

- **最小 diff**：只改与当前任务相关的文件；禁止夹带大范围格式化、无关重命名或「顺手清理」。
- 与架构边界冲突的重构：先立 issue/文档记录，再分步提交，避免单 PR 难以审查。

## 提交信息

- 使用 **简体中文完整句**，说明「做了什么、为何必要」；避免无意义的单字信息。

## 勿提交（与 DEVELOPMENT 一致）

- 含本机绝对路径的运行时快照（如 `model_repo/data/models.json`）。
- 密钥、`.env`、大体积模型与缓存目录、日志垃圾。
- **SQLite `app.db`**：是否入库由团队约定；生产数据迁移以拷贝或 `DATABASE_URL` 为准，勿误传个人调试库。详见 [DEVELOPMENT.md](../../DEVELOPMENT.md)「最终环境参数清单」「勿提交」表。

## 联调与自测

- 关键链路：**标准融合、评测、完全融合（进化）** 的验证应以 **真实 HTTP API**（或文档中约定的等价手工步骤）为准，避免仅依赖一次性本地脚本作为唯一验收。
- 具体命令与端口以 [DEVELOPMENT.md](../../DEVELOPMENT.md)「快速核验清单」及环境章节为准，本文件不重复易过时命令。
