# AI 与自动化工具协作规则（mergeKit_beta）

适用于 Cursor / Codex / 其他编码代理及使用说明驱动工具的场景。

## 「95% 把握」原则（项目强制）

**对需求边界、数据路径、兼容策略、是否触及双轨入口（如 `app.py` vs `create_app`）、破坏性操作等存在不确定或疑问时，必须先向用户/责任人追问澄清，直至对「要实现的目标、约束条件、验收方式」有至少约 **95%** 把握后，再制定可执行方案或动手改代码/改配置。**

禁止在关键信息缺失时，仅靠猜测「替用户做决定」并直接落地大改动。

## 计划与执行分界

- **计划阶段**：范围、风险、回滚、影响文件列表尚不明确 → 适用 95% 原则，先问再写方案。
- **执行阶段**：方案已确认后，仍遇未覆盖的破坏性分支 → 再次确认，不得默认执行。

## 破坏性操作须二次确认

以下操作必须经过用户/责任人明确同意（与 [DO_NOT_DELETE_FILES.md](DO_NOT_DELETE_FILES.md) 一致）：

- 删除、移动、批量清理仓库文件或目录。
- 变更数据库迁移策略、对生产库执行不可逆迁移。
- `git push --force`（尤其共享分支）或其他改写历史的 Git 操作。
- **对外 HTTP API** 的语义或兼容性破坏（路径、字段、状态码约定）。

## 沟通语言

- 与用户对话时遵循项目约定（例如 **简体中文**）。
