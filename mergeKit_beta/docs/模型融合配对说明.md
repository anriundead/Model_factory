# 模型融合配对说明

## 能否配对融合的判断依据

两个或多个模型能否进行 TIES-DARE 等融合，取决于它们是否属于**同一架构**。同一架构的判定以模型目录下 `config.json` 中的以下字段为准：

- **`hidden_size`**：隐藏层维度，必须一致。
- **`num_hidden_layers`**：隐藏层数，必须一致。

若上述两项均相同，则视为同一架构，可以配对融合；若任一项不同，则视为不同架构，不可融合。

## 为何不用 model_type 单独判断

不同来源或命名的模型可能使用不同的 `model_type`（如 `llama`、`llama3`、`Llama3`），但底层架构可能相同（hidden_size、num_hidden_layers 一致）。以 `hidden_size` 和 `num_hidden_layers` 为准，可以避免误判，并支持同架构不同命名/版本的模型互相融合。

## 实现说明

- 后端在「完全融合」前会解析每个模型路径下的 `config.json`，读取 `hidden_size` 与 `num_hidden_layers`。
- 仅当所有待融合模型的这两项完全一致时，才允许提交融合任务；否则返回不可融合原因，前端弹窗提示用户。
